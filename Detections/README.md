# Detection Engineering

## Overview
This collection showcases my ability to write detection rules across different formats (Sigma, KQL, SPL).

## Categories
- **Sigma**: Platform-agnostic rules.
- **KQL**: Azure Sentinel / Defender for Endpoint queries.
- **SPL**: Splunk Search Processing Language queries.

## Methodology
All rules follow the **Detection Development Lifecycle**:
1.  Identify TTP (MITRE ATT&CK).
2.  Simulate Attack.
3.  Draft Rule.
4.  Tune for False Positives.
5.  Deploy.
